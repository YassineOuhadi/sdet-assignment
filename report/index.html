Project Overview

Complete SDET-level workflow for Bloomberg FX deals, demonstrating CSV parsing, schema-driven validation, deduplication, partial saves, proper logging, and robust exception handling. Built with Spring Boot, Maven, Docker, and modern testing tools like JUnit, Mockito, RestAssured, and TestContainers.

Tech Stack

Java 17 Spring Boot JaCoCo JUnit Mockito AssertJ Surefire Failsafe OpenCSV SLF4J / Logback Docker Makefile Postman K6

Key Features / Flow

CSV Parsing & Validation

Automatic parsing and schema-driven validation for CSV files.

Deduplication Logic

Prevents duplicate entries and ensures data integrity.

Partial Save (No Rollback)

Saves valid rows even if some fail validation, ensuring partial data persistence.

Logging & Exception Handling

Structured logging with SLF4J and comprehensive exception handling.

Persistence & Concurrency

Supports multi-threaded processing and database transactions consistently.

Clean Architecture

Controller, Parser, Validator, and Service layers for maintainable code.

Unit Tests

JUnit & Mockito for verifying individual components in isolation.

Integration Tests

Full system tests with TestContainers, RestAssured, WebMvc, and JUnit.

Reporting & Coverage

Unit & integration reports generated with Surefire, Failsafe, and JaCoCo.
